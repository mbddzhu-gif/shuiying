# 批量加水印功能实施计划

## 目标
在现有单图加水印应用基础上，增加批量加水印功能，支持同时对多个图片文件添加自定义水印，提供清晰的文件上传区域、水印参数配置面板及批量处理进度显示。

## 架构设计

### 核心思路
- 将现有单图模式与批量模式整合在同一页面，通过 Tab 切换
- 提取水印绘制逻辑为可复用函数，单图和批量模式共用
- 批量模式使用 JSZip 库将多张处理后的图片打包为 ZIP 下载
- 批量处理采用异步逐张处理，显示实时进度条

### 文件变更清单

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/App.vue` | 重写 | 整合单图/批量双模式，提取公共水印逻辑 |
| `src/style.css` | 修改 | 重置为适合水印工具的全局样式 |
| `index.html` | 修改 | 更新页面标题 |
| `package.json` | 修改 | 添加 jszip 依赖 |

## 详细实施步骤

### 步骤 1：安装 JSZip 依赖
- 执行 `npm install jszip` 用于批量导出时打包 ZIP 文件

### 步骤 2：重写 `src/App.vue`
整体结构分为三个区域：
1. **顶部**：标题 + 模式切换 Tab（单图模式 / 批量模式）
2. **左侧**：预览区域
3. **右侧**：控制面板

#### 2.1 提取公共水印绘制函数
```ts
function drawWatermarkToCanvas(
  canvas: HTMLCanvasElement,
  img: HTMLImageElement,
  options: {
    text: string;
    size: number;
    opacity: number;
    color: string;
    spacingX: number;
    spacingY: number;
    angle: number;
  }
): void
```
- 单图预览和批量导出共用此函数
- 在 canvas 上绘制原图 + 铺满水印

#### 2.2 单图模式（保留现有功能）
- 单张图片上传
- 实时预览（图片 + 水印叠加层）
- 单张导出

#### 2.3 批量模式（新增功能）
- **文件上传区域**：支持多文件选择 + 拖拽上传，显示已选文件列表（缩略图 + 文件名 + 大小），支持删除单个文件
- **预览区域**：显示当前选中图片的预览（带水印效果），可点击文件列表切换预览
- **水印参数配置**：与单图模式共用同一套参数（文字、大小、颜色、透明度、密度、角度）
- **批量处理进度**：进度条 + 当前处理数/总数 + 处理状态文字
- **批量导出**：点击"批量导出"按钮后，逐张处理所有图片，完成后自动下载 ZIP 包

#### 2.4 批量处理流程
1. 用户上传多张图片 → 显示文件列表
2. 点击"批量导出" → 创建 JSZip 实例
3. 逐张处理：创建离屏 canvas → 绘制原图 + 水印 → 转为 Blob → 添加到 ZIP
4. 每处理一张更新进度条
5. 全部完成 → 生成 ZIP 并下载

### 步骤 3：修改 `src/style.css`
- 移除 Vite 默认模板样式
- 添加适合水印工具的全局基础样式（box-sizing、字体等）

### 步骤 4：修改 `index.html`
- 更新 `<title>` 为"图片加水印工具"

### 步骤 5：构建验证
- 执行 `npm run build` 确保无 TypeScript 错误
- 启动 `npm run dev` 进行本地测试

### 步骤 6：本地全面测试
- 单图模式：上传 JPG/PNG/WebP，调整各参数，导出验证
- 批量模式：上传多张不同格式图片，调整参数，批量导出 ZIP 验证
- 边界条件：空文字、最大/最小参数值、大量图片（20+张）性能

### 步骤 7：推送到 GitHub
- 初始化 Git 仓库（如尚未初始化）
- 提交所有代码
- 推送到 GitHub 远程仓库，触发 Vercel 自动部署
